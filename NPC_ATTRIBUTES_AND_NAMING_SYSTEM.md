## 1. NPC 属性模板设计

为了确保 200 个 NPC 具有深度和一致性，我们将采用统一的属性模板。每个 NPC 都将拥有以下核心属性：

### 1.1 核心属性结构

```json
{
  "npcId": "string",            // 唯一标识符，格式: [职业缩写]_[编号]，如: BANK_MGR_001
  "name": "string",             // 中文名，如: 艾琳·雪莱
  "alias": "string",            // 别名/昵称 (可选)
  "gender": "string",           // 性别: "男", "女", "未知"
  "ageRange": "string",         // 年龄范围: "青年", "中年", "老年"
  "profession": "string",       // 职业，如: "银行行长", "投资顾问", "咖啡师"
  "socialClass": "string",      // 社会阶层: "上层", "中层", "下层"
  "personalityTraits": [        // 个性特征，至少 3 个，如: "严谨", "专业", "有远见"
    "string"
  ],
  "backgroundStory": "string",  // 背景故事，约 50-100 字
  "economicRole": "string",     // 在 ISC 经济中的具体作用，如: "管理银行运营、提供金融服务"
  "skills": [                   // 技能列表 (可选)
    "string"
  ],
  "hobbies": [                  // 爱好列表 (可选)
    "string"
  ],
  "relationshipStatus": "string", // 关系状态: "单身", "已婚", "未知"
  "currentLocation": "string",  // 当前所在区域/地点
  "dailyRoutine": "string",     // 每日作息描述
  "dialogueStyle": "string",    // 对话风格描述，如: "正式、简洁、数据导向"
  "visualDescription": "string",// 详细视觉描述，用于 AI 图像生成
  "aiImagePrompt": "string"     // AI 图像生成提示词
}
```

### 1.2 属性详细说明

-   **npcId**: 采用 `[职业缩写]_[编号]` 的格式，例如 `BANK_MGR_001` (银行经理 001)。确保唯一性和可读性。
-   **name**: 重新设计的中文名，将采用更具文化内涵和角色特点的命名方式。
-   **profession**: 参照 `NPC_VISUAL_DESIGN_AND_CLASSIFICATION.md` 中的职业分类。
-   **socialClass**: 参照 `NPC_VISUAL_DESIGN_AND_CLASSIFICATION.md` 中的社会阶层分类。
-   **personalityTraits**: 参照 `NPC_VISUAL_DESIGN_AND_CLASSIFICATION.md` 中的个性特征分类，并可扩展。
-   **backgroundStory**: 简短精炼，突出 NPC 的核心经历和动机。
-   **economicRole**: 明确 NPC 在游戏经济中的具体功能和贡献。
-   **visualDescription**: 详细描述 NPC 的外貌、服装、配饰等，作为 AI 图像生成的依据。
-   **aiImagePrompt**: 根据 `visualDescription` 提炼出的 AI 图像生成提示词，确保生成图像符合设计要求。

## 2. NPC 名字系统重新设计

针对之前“扯淡的中文 NPC 名字”的反馈，我们将采用一套更具深度、文化内涵和角色匹配度的命名系统。新名字将避免过于直白或带有刻板印象的称谓，转而使用更具文学性和想象力的组合。

### 2.1 命名原则

-   **文化内涵**: 名字应具有一定的文化底蕴或意境，避免过于口语化。
-   **角色匹配**: 名字应与 NPC 的职业、个性、背景故事相符，增强角色的辨识度。
-   **音韵优美**: 名字读起来应流畅悦耳，易于记忆。
-   **多样性**: 确保 200 个 NPC 的名字具有足够的多样性，避免重复或雷同。
-   **冰雪主题**: 可适当融入冰雪、寒冷、晶莹、纯洁等主题元素，但不过度。
-   **全球化考量**: 名字应具有一定的普适性，不局限于单一地域文化。

### 2.2 命名结构建议

我们将采用 **姓氏 + 名字** 或 **名字 + 别名/称号** 的组合方式，并可引入西方姓氏或中文姓氏的组合，以体现全球化背景。

#### 2.2.1 姓氏 + 名字 (常用)

-   **中文姓氏**: 如“林”、“陈”、“王”、“李”、“张”等常见姓氏。
-   **西方姓氏**: 如“雪莱 (Shelley)”、“艾伦 (Allen)”、“维克 (Vick)”等，可与中文名组合。
-   **名字**: 选用具有意境或美好寓意的字词，如“清”、“澜”、“澈”、“霜”、“月”、“星”、“辰”、“曦”、“岚”等。

**示例**:
-   **银行行长**: 艾琳·雪莱 (Eileen Shelley) - 优雅、专业
-   **投资顾问**: 维克·辰 (Vick Chen) - 精明、敏锐
-   **咖啡师**: 林·霜月 (Lin Shuangyue) - 温和、细腻
-   **工程师**: 张·云澜 (Zhang Yunlan) - 沉稳、智慧
