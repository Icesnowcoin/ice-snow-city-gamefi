# Ice Snow City - NPC 关系网络设计

## 简介

本文档定义了 Ice Snow City 游戏中 NPC 之间的关系网络系统，以及这些关系如何与经济系统和原型库集成。

---

## 1. NPC 关系类型

### 1.1 亲属关系
- **父子关系**: NPC A 是 NPC B 的父亲/母亲
- **兄弟姐妹关系**: NPC A 和 NPC B 是兄弟姐妹
- **配偶关系**: NPC A 和 NPC B 是夫妻

### 1.2 职业关系
- **上下级关系**: NPC A 是 NPC B 的上司/下属
- **同事关系**: NPC A 和 NPC B 在同一机构工作
- **合作伙伴关系**: NPC A 和 NPC B 进行商业合作

### 1.3 社交关系
- **朋友关系**: NPC A 和 NPC B 是朋友
- **敌对关系**: NPC A 和 NPC B 存在冲突
- **竞争关系**: NPC A 和 NPC B 在竞争同一资源或市场

### 1.4 经济关系
- **债务关系**: NPC A 欠 NPC B ISC
- **投资关系**: NPC A 投资于 NPC B 的项目
- **雇佣关系**: NPC A 雇佣 NPC B 进行工作

## 2. NPC 关系网络结构

### 2.1 数据库表设计

```sql
CREATE TABLE npc_relationships (
  id INT PRIMARY KEY AUTO_INCREMENT,
  npc_id_1 VARCHAR(50) NOT NULL,
  npc_id_2 VARCHAR(50) NOT NULL,
  relationship_type VARCHAR(50) NOT NULL,
  relationship_strength INT DEFAULT 50,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY unique_relationship (npc_id_1, npc_id_2, relationship_type),
  FOREIGN KEY (npc_id_1) REFERENCES npc_profiles(npc_id),
  FOREIGN KEY (npc_id_2) REFERENCES npc_profiles(npc_id)
);
```

### 2.2 关系强度

关系强度范围从 0 到 100：
- **0-20**: 非常弱的关系（几乎不认识）
- **21-40**: 弱的关系（认识但不熟悉）
- **41-60**: 中等关系（熟悉）
- **61-80**: 强的关系（亲密）
- **81-100**: 非常强的关系（极其亲密）

## 3. NPC 关系与经济系统的集成

### 3.1 关系影响 ISC 交易

**亲密度奖励**:
- 与关系强度高的 NPC 交易时，获得 5-10% 的 ISC 奖励
- 与关系强度低的 NPC 交易时，可能需要支付 2-5% 的额外费用

**关系发展**:
- 每次与 NPC 交易，关系强度增加 1-2 点
- 每月不交互，关系强度减少 1 点

### 3.2 关系影响任务奖励

**任务难度调整**:
- 与关系强度高的 NPC 的任务难度降低 10%，奖励增加 15%
- 与关系强度低的 NPC 的任务难度增加 10%，奖励减少 5%

**任务可用性**:
- 只有关系强度 > 40 的 NPC 才会提供特殊任务
- 只有关系强度 > 70 的 NPC 才会提供高级任务

### 3.3 关系影响 NPC 经济行为

**投资决策**:
- NPC 更可能投资于关系强度高的其他 NPC 的项目
- NPC 更可能从关系强度高的其他 NPC 购买商品

**价格协商**:
- 与关系强度高的 NPC 交易时，价格更优惠
- 与关系强度低的 NPC 交易时，价格更昂贵

## 5. 关系网络与原型库的集成

### 5.1 原型库中的关系信息

每个 NPC 原型应包含以下关系信息：

```json
{
  "npcId": "NPC_001",
  "name": "李·行长",
  "relationships": [
    {
      "targetNpcId": "NPC_002",
      "relationshipType": "上下级",
      "relationshipStrength": 75,
      "description": "工作上的直接下属"
    },
    {
      "targetNpcId": "NPC_003",
      "relationshipType": "同事",
      "relationshipStrength": 80,
      "description": "长期的合作伙伴"
    }
  ]
}
```

### 5.2 关系网络初始化

在游戏启动时，系统应：

1. 从 NPC 原型库加载所有 NPC 的关系信息
2. 将关系信息写入 `npc_relationships` 表
3. 初始化关系强度值
4. 建立关系索引以加快查询

### 5.3 关系网络的动态更新

游戏运行过程中，系统应：

1. 监听 NPC 交易事件，更新关系强度
2. 监听 NPC 任务完成事件，更新关系强度
3. 定期检查关系衰减，更新关系强度
4. 记录关系变化的审计日志

## 6. NPC 关系网络的可视化

### 6.1 关系图表示

在游戏 UI 中，可以使用力导向图（Force-Directed Graph）来可视化 NPC 关系网络：

- **节点**: 代表 NPC
- **边**: 代表关系
- **边的粗细**: 代表关系强度
- **边的颜色**: 代表关系类型

### 6.2 关系查询接口

玩家可以通过以下接口查询 NPC 关系：

- `getNPCRelationships(npcId)`: 获取指定 NPC 的所有关系
- `getRelationshipStrength(npcId1, npcId2)`: 获取两个 NPC 之间的关系强度
- `getRelationshipPath(npcId1, npcId2)`: 获取两个 NPC 之间的关系路径

## 7. 关系网络对游戏经济的影响

### 7.1 ISC 流动

关系网络影响 ISC 在 NPC 之间的流动：

- 强关系的 NPC 之间的交易更频繁
- 弱关系的 NPC 之间的交易成本更高
- 敌对关系的 NPC 之间几乎不进行交易

### 7.2 经济稳定性

关系网络有助于维持游戏经济的稳定性：

- 紧密的关系网络可以防止经济崩溃
- 孤立的 NPC 可能导致经济波动
- 关系多样化可以提高经济韧性

### 7.3 玩家参与

玩家可以通过影响 NPC 关系来影响游戏经济：

- 与特定 NPC 建立关系可以获得经济优势
- 破坏 NPC 关系可能导致经济风险
- 建立新的关系可以创造新的经济机会

---

## 8. 总结

NPC 关系网络是 Ice Snow City 游戏经济系统的重要组成部分。通过精心设计的关系系统，游戏可以创造一个动态、真实的虚拟经济生态，其中 NPC 之间的互动和关系直接影响游戏的经济流动和玩家的游戏体验。
