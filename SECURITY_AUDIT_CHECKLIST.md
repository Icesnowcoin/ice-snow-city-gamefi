# Ice Snow City - 安全审计检查清单

## 1. 身份认证与授权

### 1.1 密码安全
- [ ] 所有密码使用强哈希算法（Keccak256 或 bcrypt）
- [ ] 密码最小长度要求 ≥ 12 个字符
- [ ] 实现密码过期策略（90 天）
- [ ] 禁止使用常见密码
- [ ] 实现账户锁定机制（5 次失败后锁定 30 分钟）

### 1.2 会话管理
- [ ] 会话 Token 使用 JWT 或安全 Session Cookie
- [ ] Token 过期时间设置合理（≤ 24 小时）
- [ ] 实现 Token 刷新机制
- [ ] 支持会话注销和 Token 撤销
- [ ] 使用 HttpOnly 和 Secure 标志的 Cookie

### 1.3 访问控制
- [ ] 实现基于角色的访问控制（RBAC）
- [ ] 所有者级别的特殊权限验证
- [ ] 常数时间密钥验证（防止时序攻击）
- [ ] 审计所有权限变更

## 2. 数据保护

### 2.1 传输安全
- [ ] 使用 HTTPS/TLS 1.2+ 加密所有通信
- [ ] 实现 HSTS（HTTP Strict Transport Security）
- [ ] 禁用不安全的 HTTP 方法
- [ ] 实现 CORS 策略

### 2.2 数据存储
- [ ] 敏感数据加密存储（AES-256）
- [ ] 数据库连接使用 SSL/TLS
- [ ] 定期备份加密
- [ ] 备份存储在安全位置（S3 + 加密）

### 2.3 敏感信息处理
- [ ] 不记录密钥、密码、Token 等敏感信息
- [ ] 日志中敏感数据部分遮蔽
- [ ] 实现日志保留策略（≤ 90 天）
- [ ] 审计日志不可篡改

## 3. 输入验证与输出编码

### 3.1 输入验证
- [ ] 所有输入进行类型检查
- [ ] 实现长度限制
- [ ] 验证数据格式（邮箱、URL 等）
- [ ] 防止 SQL 注入（使用参数化查询）
- [ ] 防止 NoSQL 注入

### 3.2 输出编码
- [ ] 正确编码 HTML 输出
- [ ] JSON 响应设置正确的 Content-Type
- [ ] 防止 XSS 攻击
- [ ] 防止 CSRF 攻击（CSRF Token）
